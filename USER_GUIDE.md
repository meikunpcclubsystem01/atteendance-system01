# 🏫 自習室入退室管理システム：ユーザーマニュアル (全利用者版)

このガイドは、自習室を安全かつ便利に活用するための全ての情報を網羅しています。
**生徒さん、保護者のみなさま、そして運営を担当する管理者（パソコン部・当番）**それぞれに向けた詳細なステップバイステップを解説します。

---

## 1. 生徒のみなさんへ：日常の使いかた (Student Guide)

自習室の入退室は、学習時間の記録でもあります。正しく使って、自分の頑張りを確認しましょう！

### 1.1 ログインと初回セットアップ
- **URL**: [https://entry.meikun-pc.club](https://entry.meikun-pc.club)
  
  ![システムへのアクセス用QRコード](https://api.qrserver.com/v1/create-qr-code/?size=150x150&data=https://entry.meikun-pc.club/)
  
- **アカウント**: 学校から配られた **@niigata-meikun.ed.jp** のアカウントでGoogleログインしてください。
- **初回登録**: 名前、学籍番号、保護者の方のメールアドレスを正しく入力。これが間違っていると通知が届きません！

### 1.2 QRコードの提示 (Check-in / Check-out)
- **スキャン手順**: ダッシュボードに表示されるQRコードを受付のカメラにかざします。
- **スクショは無効**: QRコードは **30秒ごとに新しくなります**。さらに「動く枠線」と「動く時計」が本物の画面である証拠です。
- **「ピッ」の音**: 成功すると画面に反映されます。「在室中」になったことを確認して席につきましょう。

### 1.3 座席の選びかた
- 入室時は、受付の画面で好きな空席を選んでください。
- **ダブルブッキング防止**: 他の人が選んだ瞬間の座席は選べないようになっています。

### 1.4 利用申請 (保護者への承認依頼)
- 休暇中や新年度などで「利用期間外」と出た場合は、画面のボタンから保護者に申請を送れます。
- **コツ**: 申請を送ったら、保護者の方に「メールが届いているから承認して！」と直接伝えるとスムーズです。

## 2. 保護者のみなさまへ：安全の見守り (Parent Guide)

お子さまの安全と学習環境を守るための、大切な操作について解説します。

### 2.1 入退室のリアルタイム通知
- お子さまが「入室したとき」と「退出したとき」、即座にメールが届きます。
- **履歴の確認**: メール内の「利用状況を確認する」リンクから、過去30日間の学習履歴を見ることができます。

### 2.2 利用の承認 (マジックリンク方式)
- お子さまから申請があると、承認用メールが届きます。
- **操作**: メール内のボタンを押すだけで承認完了です。
- **安全性**: パスワード入力やログインは不要ですが、一度使ったリンクは物理的に使えなくなる（再利用防止）ため、お子さまが勝手に操作することはできません。

## 3. 管理・運営スタッフのみなさんへ (Admin Guide)

自習室の快適な環境を維持するための、管理者向け操作です。

### 3.1 受付（スキャナー）の設置方法
- 受付の専用端末（またはPC）で `/scanner` ページを開きます。
- **ブラウザの全画面表示 (F11)** を推奨。
- **カメラの角度**: QRコードを読み取りやすいよう、少し手前に向けて設置します。

### 3.2 異常時の対応 (手動ログアウト)
- 「帰ったはずの生徒がまだ在宅のままになっている（座席が埋まっている）」場合：
  1. 管理画面の「在室者一覧」を開く。
  2. 該当する生徒の右側にある **「強制退出」** ボタンを押す。
  3. これで座席が開放され、ログにも「管理者による処理」として残ります。

### 3.3 座席レイアウトの現場編集
- 「自習室の模様替え」をした際は、管理画面から座席表を書き換えられます。
- **カンマ区切り**: `A1, A2, A3` のように並べることで、即座に受付画面に反映されます。

## 🛠 管理者（運営スタッフ）のみなさんへ：用語ミニ解説

管理画面で見かける「ちょっと難しい言葉」の解説です。

- **ログ (Log)**: 「いつ・誰が・何をしたか」の記録のこと。
- **强制退出 (Force Logout)**: 帰り忘れた生徒の在室状態を、管理者が手動で「退室」に変えること。
- **レイアウト (Layout)**: 座席の並び順や配置のこと。
- **ステータス (Status)**: 現在の状態（「在室中」か「退室済」か）のこと。
- **ダッシュボード (Dashboard)**: システム全体の状況を一目で確認できる管理のトップ画面。

## 4. よくある質問とトラブル解決 (Advanced FAQ)

- **Q. カメラがQRコードを全然読み取りません。**
  - **原因**: 部屋の照明が反射している、またはスマホの画面が暗すぎます。
  - **対策**: スマホの輝度を最大にし、カメラに対して少し斜めにかざしてみてください。
- **Q. ログインしようとすると「このドメインは許可されていません」と出ます。**
  - **原因**: 個人のGmailでログインしようとしています。
  - **対策**: 一度 Google をログアウトし、学校のアカウントでログインし直してください。
- **Q. 保護者にメールが届きません。**
  - **原因**: 迷惑メールフォルダに入っている、または登録アドレスのスペルミス。
  - **対策**: 管理者に頼んで、名簿から登録アドレスを修正してもらってください。
